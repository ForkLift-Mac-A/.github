## 🔍 Overview

<a href="#forkLift">ForkLift</a> is a dual-pane file manager and remote connection client for Mac — providing the side-by-side file navigation, remote server access, and powerful file operation capabilities that power users, developers, and system administrators need beyond what Finder's single-pane browsing provides.

The <a href="#forkLift">dual-pane interface</a> is the defining characteristic of ForkLift's file management approach — two file browser panels display simultaneously, making file operations between locations immediate and visual. Copying files from a local folder to an FTP server, moving files between two local directories, or synchronizing folders between local and remote locations all benefit from seeing both source and destination simultaneously rather than navigating back and forth in a single-pane browser. The <a href="#forkLift">remote connection protocol support</a> covers FTP, SFTP, FTPS, WebDAV, Amazon S3, Backblaze B2, SMB, NFS, AFP, and other protocols — connecting to web servers, network-attached storage, cloud storage services, and corporate file shares from the same ForkLift interface that manages local files. Remote server connections appear in the file browser exactly like local folders, enabling the same drag-and-drop operations between local and remote locations.

The <a href="#forkLift">folder synchronization</a> compares two directories — local and remote, or two local directories — and provides a detailed view of which files differ, which are new, and which have been deleted, with control over which changes to apply in which direction. Synchronizing a local project folder with a server deployment folder, keeping a backup folder current, or maintaining consistency between two locations becomes a precise, reviewable operation rather than a blind copy. The <a href="#forkLift">advanced search and file finding</a> searches by name, content, size, date, and other criteria across local and remote locations simultaneously — finding files regardless of which connected location they live in.</p>

<a href="#forkLift">Multi-rename tool</a> applies batch renaming operations to groups of files — adding prefixes, removing suffixes, changing case, replacing text strings, inserting sequential numbers, and applying any combination of renaming rules to hundreds of files simultaneously. <a href="#forkLift">Drop zones</a> provide quick-access destination shortcuts — defining frequently used local and remote locations as drop zone targets for immediate drag-and-drop access without navigating to the destination first.

<a href="#forkLift">Terminal integration</a> opens a terminal window in the current ForkLift directory — switching between the graphical file manager and terminal command line for the same location without navigating there again in the terminal. <a href="#forkLift">Quick Look file preview</a> renders the macOS Quick Look preview for any file in the ForkLift browser — inspecting file content without opening the file in its application, including remote files over supported connection types.</p>

## 💬 FAQ

<details>
<summary>What remote protocols does ForkLift support?</summary>

FTP, SFTP, FTPS, WebDAV, Amazon S3, Backblaze B2, SMB, NFS, AFP, and other connection types. Check current ForkLift documentation for the complete protocol list.

</details>

<details>
<summary>What is the dual-pane interface?</summary>

ForkLift shows two file browser panels simultaneously — source on one side, destination on the other — making file operations between two locations visual and immediate.

</details>

<details>
<summary>Does ForkLift synchronize folders?</summary>

Yes. Folder sync compares two directories, shows what differs, and provides control over which changes to apply in which direction.

</details>

<details>
<summary>Can ForkLift batch rename files?</summary>

Yes. The multi-rename tool applies combinations of renaming rules — prefixes, suffixes, sequential numbers, text replacement — to groups of files simultaneously.

</details>

<details>
<summary>Does ForkLift integrate with Terminal?</summary>

Yes. ForkLift opens a terminal window at the current browser directory for immediate command-line access without separate navigation.

</details>

<details>
<summary>Does ForkLift have Quick Look support?</summary>

Yes. macOS Quick Look previews any file in the ForkLift browser, including files on remote connections where supported.

</details>

<details>
<summary>Can ForkLift compare two files?</summary>

Yes. The file comparison tool shows line-by-line differences between two files side by side.

</details>

<details>
<summary>What are Drop Zones?</summary>

Drop Zones are saved location shortcuts — quick-access targets for dragging files to frequently used local and remote destinations without navigating there first.

</details>

<details>
<summary>Does ForkLift run on Apple Silicon?</summary>

Yes. Current versions support Apple Silicon Macs natively.

</details>
